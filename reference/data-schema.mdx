Every Tododeia report is stored as a JSON file conforming to the interfaces defined in **`dashboard/src/types/report.ts`**. This page documents every field across all interfaces.

## File locations

| File | Description |
|------|-------------|
| **`dashboard/public/data/report.json`** | English report (written each run) |
| **`dashboard/public/data/report-es.json`** | Spanish report (written each run) |
| **`output/history/YYYY-MM-DD.json`** | Dated archive (last 30 runs kept) |

## Top-level structure

The following JSON shows the full top-level shape of a report file. Nested objects are shown with ellipses.

```json
{
  "brand": "Tododeia",
  "creator": "@soyenriquerocha",
  "generated_at": "2026-03-18T14:30:00Z",
  "risk_profile": "moderate",
  "executive_summary": "...",
  "macro_environment": { ... },
  "portfolio_allocation": { ... },
  "cross_sector_insights": [ ... ],
  "risk_adjusted_picks": [ ... ],
  "historical_accuracy": { ... },
  "warnings": [ ... ],
  "sectors": {
    "crypto": { ... },
    "stocks": { ... },
    "currencies": { ... },
    "materials": { ... }
  }
}
```

## ReportData

The root object of every report file.

<ResponseField name="brand" type="string" required>
  Always `"Tododeia"`. Identifies the report source.
</ResponseField>

<ResponseField name="creator" type="string" required>
  Always `"@soyenriquerocha"`. The skill author.
</ResponseField>

<ResponseField name="generated_at" type="string" required>
  ISO 8601 timestamp of when the report was generated (e.g., `"2026-03-18T14:30:00Z"`).
</ResponseField>

<ResponseField name="risk_profile" type="string" required>
  The risk tolerance selected by the user. One of: `"conservative"`, `"moderate"`, `"aggressive"`.
</ResponseField>

<ResponseField name="executive_summary" type="string" required>
  A 3–4 sentence strategy overview produced by the Strategy Agent, tailored to the selected risk profile.
</ResponseField>

<ResponseField name="macro_environment" type="MacroEnvironment" required>
  Macro-level assessment of interest rates, inflation, and geopolitical risk. See [MacroEnvironment](#macroenvironment).
</ResponseField>

<ResponseField name="portfolio_allocation" type="PortfolioAllocation" required>
  Recommended percentage allocation per sector. Keys are sector names; values are integers that sum to 100. See [PortfolioAllocation](#portfolioallocation).
</ResponseField>

<ResponseField name="cross_sector_insights" type="CrossSectorInsight[]" required>
  List of cross-sector correlations or divergences identified by the Strategy Agent. See [CrossSectorInsight](#crosssectorinsight).
</ResponseField>

<ResponseField name="risk_adjusted_picks" type="RiskAdjustedPick[]" required>
  Ranked list of the top investment picks across all sectors, adjusted for the user's risk profile. See [RiskAdjustedPick](#riskadjustedpick).
</ResponseField>

<ResponseField name="historical_accuracy" type="HistoricalAccuracy" required>
  Accuracy metrics comparing previous recommendations to actual price outcomes. See [HistoricalAccuracy](#historicalaccuracy).
</ResponseField>

<ResponseField name="warnings" type="string[]" required>
  List of risk warnings or cautions generated by the Strategy Agent.
</ResponseField>

<ResponseField name="sectors" type="Record<string, SectorData>" required>
  Map of sector name to sector data. Keys are `"crypto"`, `"stocks"`, `"currencies"`, and `"materials"`. See [SectorData](#sectordata).
</ResponseField>

## MacroEnvironment

Macro-level analysis synthesized by the Strategy Agent from all four sector reports.

<ResponseField name="summary" type="string" required>
  A 2–3 sentence narrative overview of the current macro environment (rates, inflation, geopolitics).
</ResponseField>

<ResponseField name="interest_rate_outlook" type="string" required>
  Expected direction of interest rates. One of: `"rising"`, `"stable"`, `"falling"`.
</ResponseField>

<ResponseField name="inflation_outlook" type="string" required>
  Expected direction of inflation. One of: `"rising"`, `"stable"`, `"falling"`.
</ResponseField>

<ResponseField name="geopolitical_risk" type="string" required>
  Overall geopolitical risk level. One of: `"high"`, `"medium"`, `"low"`.
</ResponseField>

<ResponseField name="key_factors" type="string[]" required>
  List of the key macro drivers influencing the current environment (e.g., `"Fed expected to hold rates through Q2"`).
</ResponseField>

## PortfolioAllocation

A flat key-value map of sector names to integer percentages. All values must sum to 100.

```typescript
export type PortfolioAllocation = Record<string, number>
```

Example:

```json
{
  "crypto": 10,
  "stocks": 45,
  "currencies": 15,
  "materials": 20,
  "cash": 10
}
```

The keys and their relative weights vary based on the user's risk profile. Conservative profiles hold more cash and materials; aggressive profiles allocate more to crypto and growth stocks.

## CrossSectorInsight

A single cross-sector observation identified by the Strategy Agent.

<ResponseField name="insight" type="string" required>
  A description of the correlation or divergence observed across sectors (e.g., `"Gold and Bitcoin are both rallying simultaneously"`).
</ResponseField>

<ResponseField name="implication" type="string" required>
  What the insight means for investors (e.g., `"This suggests broad hedging against fiat devaluation, favoring hard assets"`).
</ResponseField>

## RiskAdjustedPick

A ranked top investment pick, scored and sized for the user's risk profile.

<ResponseField name="rank" type="number" required>
  Rank position (1 = highest conviction). At least 5 picks are generated per report.
</ResponseField>

<ResponseField name="name" type="string" required>
  Full asset name (e.g., `"NVIDIA"`).
</ResponseField>

<ResponseField name="symbol" type="string" required>
  Ticker or trading symbol (e.g., `"NVDA"`).
</ResponseField>

<ResponseField name="sector" type="string" required>
  The sector this asset belongs to. One of: `"crypto"`, `"stocks"`, `"currencies"`, `"materials"`.
</ResponseField>

<ResponseField name="confidence" type="number" required>
  Raw confidence score from the sector agent (1–10 scale).
</ResponseField>

<ResponseField name="risk_score" type="number" required>
  Volatility/risk score assigned by the Strategy Agent (1–10, where 10 = highest risk).
</ResponseField>

<ResponseField name="risk_adjusted_score" type="number" required>
  Final composite score after applying risk profile modifiers (e.g., `8.2`). Used to determine rank.
</ResponseField>

<ResponseField name="recommendation" type="string" required>
  Trading recommendation. One of: `"buy"`, `"hold"`, `"sell"`.
</ResponseField>

<ResponseField name="reasoning" type="string" required>
  Risk-adjusted explanation for the recommendation, written for the user's specific risk profile.
</ResponseField>

<ResponseField name="position_size" type="string" required>
  Suggested portfolio allocation for this single asset (e.g., `"8-10% of portfolio"`).
</ResponseField>

---

## HistoricalAccuracy

Accuracy metrics derived by comparing the previous report's recommendations to current prices.

<ResponseField name="previous_date" type="string | null" required>
  Date of the previous report used for comparison (format: `"YYYY-MM-DD"`). `null` on the first run when no history exists.
</ResponseField>

<ResponseField name="calls_made" type="number" required>
  Total number of buy and sell recommendations evaluated from the previous report. `0` on the first run.
</ResponseField>

<ResponseField name="calls_correct" type="number" required>
  Number of recommendations that were directionally correct. `0` on the first run.
</ResponseField>

<ResponseField name="accuracy_pct" type="number" required>
  Percentage of correct calls (`calls_correct / calls_made * 100`). `0` on the first run.
</ResponseField>

<ResponseField name="notable" type="string" required>
  A human-readable note about the best or most notable call from the previous report (e.g., `"BTC buy call at $65,000 now at $67,500 (+3.8%) — correct"`).
</ResponseField>

## SectorData

The full output from a single sector research agent.

<ResponseField name="sector" type="string" required>
  Sector identifier. One of: `"crypto"`, `"stocks"`, `"currencies"`, `"materials"`.
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  ISO 8601 timestamp of when the sector agent completed its research.
</ResponseField>

<ResponseField name="assets" type="Asset[]" required>
  List of 5–8 assets researched by this sector agent. See [Asset](#asset).
</ResponseField>

<ResponseField name="sector_summary" type="string" required>
  A 2–3 sentence narrative overview of the sector's current state.
</ResponseField>

<ResponseField name="sector_outlook" type="string" required>
  The sector agent's overall directional view. One of: `"bullish"`, `"bearish"`, `"neutral"`.
</ResponseField>

<ResponseField name="top_pick" type="string" required>
  Ticker symbol of the single best opportunity in this sector (e.g., `"BTC"`).
</ResponseField>

<ResponseField name="top_pick_reasoning" type="string" required>
  Explanation of why this asset is the top pick in this sector.
</ResponseField>

<ResponseField name="data_unavailable" type="boolean">
  Optional. Set to `true` when an agent fails to return valid data after retry. The sector is marked as unavailable rather than omitted.
</ResponseField>

## Asset

Detailed research data for a single asset, as returned by a sector agent.

<ResponseField name="name" type="string" required>
  Full asset name (e.g., `"Bitcoin"`, `"NVIDIA"`, `"EUR/USD"`, `"Gold"`).
</ResponseField>

<ResponseField name="symbol" type="string" required>
  Ticker or trading symbol (e.g., `"BTC"`, `"NVDA"`, `"EUR/USD"`, `"XAU"`).
</ResponseField>

<ResponseField name="current_price" type="string" required>
  Current price as a formatted string (e.g., `"$67,500.00"` for crypto/stocks, `"1.0850"` for currency pairs, `"$2,340.00"` for commodities).
</ResponseField>

<ResponseField name="change_24h" type="string" required>
  Percentage price change over the last 24 hours (e.g., `"+2.3%"`, `"-0.8%"`).
</ResponseField>

<ResponseField name="change_7d" type="string" required>
  Percentage price change over the last 7 days.
</ResponseField>

<ResponseField name="change_30d" type="string" required>
  Percentage price change over the last 30 days.
</ResponseField>

<ResponseField name="ytd_change" type="string" required>
  Year-to-date percentage price change.
</ResponseField>

<ResponseField name="week_52_high" type="string" required>
  Highest price reached in the trailing 52 weeks, formatted as a string.
</ResponseField>

<ResponseField name="week_52_low" type="string" required>
  Lowest price reached in the trailing 52 weeks, formatted as a string.
</ResponseField>

<ResponseField name="market_cap" type="string" required>
  Market capitalization as a formatted string (e.g., `"$1.3T"`, `"$850B"`). For currency pairs, this may reflect notional market size.
</ResponseField>

<ResponseField name="volume_24h" type="string" required>
  24-hour trading volume as a formatted string (e.g., `"$28B"`).
</ResponseField>

<ResponseField name="sentiment" type="string" required>
  Technical / price-action sentiment. One of: `"bullish"`, `"bearish"`, `"neutral"`.
</ResponseField>

<ResponseField name="social_sentiment" type="string" required>
  Sentiment from social media (Twitter/X, Reddit). One of: `"bullish"`, `"bearish"`, `"neutral"`, `"mixed"`.
</ResponseField>

<ResponseField name="social_buzz" type="string" required>
  Volume of social media discussion. One of: `"high"`, `"medium"`, `"low"`.

  - **high** — Trending on Twitter/X, high Reddit activity, mainstream media coverage
  - **medium** — Normal engagement levels, some discussion
  - **low** — Minimal social discussion, under the radar
</ResponseField>

<ResponseField name="confidence" type="number" required>
  Agent's confidence in its analysis on a 1–10 scale.

  - **8–10**: Very strong conviction — multiple confirming signals across price action, fundamentals, news, and social sentiment
  - **5–7**: Moderate conviction — some mixed signals or sources disagree
  - **1–4**: Low conviction — highly uncertain, conflicting data, or insufficient information
</ResponseField>

<ResponseField name="source_agreement" type="string" required>
  Agreement level between the 2+ sources the agent cross-referenced for price. One of: `"high"`, `"medium"`, `"low"`.

  - **high**: Sources agree within 1%
  - **medium**: Sources differ by 1–3%
  - **low**: Sources differ by more than 3%
</ResponseField>

<ResponseField name="sources_checked" type="string[]" required>
  List of domains or source names the agent consulted to verify this asset's data (e.g., `["coingecko.com", "yahoo.com", "coindesk.com"]`).
</ResponseField>

<ResponseField name="key_news" type="string[]" required>
  Recent news headlines relevant to this asset (e.g., `["ETF inflows surge to $500M daily", "Fed signals rate pause"]`).
</ResponseField>

<ResponseField name="social_highlights" type="string[]" required>
  Notable social media posts, trending topics, or influencer opinions about this asset.
</ResponseField>

<ResponseField name="recommendation" type="string" required>
  The sector agent's trading recommendation. One of: `"buy"`, `"hold"`, `"sell"`.

  - **buy**: Strong upward momentum, positive catalysts, undervalued relative to fundamentals
  - **hold**: Stable with no clear directional signal, wait for confirmation
  - **sell**: Negative momentum, regulatory risks, overbought conditions
</ResponseField>

<ResponseField name="reasoning" type="string" required>
  A 1–2 sentence explanation of the recommendation, grounded in the agent's research.
</ResponseField>
