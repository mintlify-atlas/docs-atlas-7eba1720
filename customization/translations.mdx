---
title: "Adding and modifying translations"
description: "How to add new UI translations or modify existing ones in the dashboard."
---

The dashboard has two separate translation layers that work together:

1. **UI translations** in **`dashboard/src/lib/translations.ts`** — static strings for buttons, labels, column headers, and other interface chrome. You edit this file manually.
2. **Report data translation** — human-readable content inside the JSON report (summaries, reasoning, news headlines, insights, warnings). This is translated automatically by the Translation Agent at runtime (Step 8b in SKILL.md).

<Tip>
Keep `translations.ts` for UI chrome only — section titles, button labels, table headers, error messages, and similar static text. Leave all report data (asset reasoning, market summaries, key news) to the Translation Agent, which handles those fields automatically.
</Tip>

## The UI translations structure

The file exports a `TRANSLATIONS` object keyed by language code (`"en"` or `"es"`), with dot-separated keys grouped by UI section:

```typescript
export type Language = "en" | "es"

export const TRANSLATIONS: Record<Language, Record<string, string>> = {
  en: {
    // Header
    "header.subtitle": "Investment Analysis by",
    "header.report": "Investment Analysis Report",
    "header.profile": "profile",

    // Toolbar
    "toolbar.filter": "Filter",
    "toolbar.buy": "Buy",
    "toolbar.hold": "Hold",
    "toolbar.sell": "Sell",
    "toolbar.search": "Search assets...",
    "toolbar.print": "Print / PDF",

    // Top Picks
    "picks.title": "Risk-Adjusted Top Picks",
    "picks.empty": "No picks match the current filter.",
    "picks.riskAdj": "Risk-Adj Score",
    "picks.confidence": "Confidence",

    // Historical Accuracy
    "accuracy.title": "Historical Accuracy",
    "accuracy.empty": "No historical data yet. Accuracy tracking begins after your second report.",

    // Disclaimer
    "disclaimer.text": "This report is generated by an AI-powered multi-agent research system...",

    // Error
    "error.noData": "No report data found. Run the investment analysis skill to generate a report.",

    // Sector names
    "sector.crypto": "Cryptocurrency",
    "sector.stocks": "Stock Market",
    "sector.currencies": "Forex & Currencies",
    "sector.materials": "Commodities & Materials",

    // ... and more
  },
  es: {
    "header.subtitle": "Análisis de Inversión por",
    "header.report": "Reporte de Análisis de Inversión",
    // ... Spanish equivalents for every key
  }

```

The full file contains keys organized into these groups: `header`, `toolbar`, `executive`, `macro`, `allocation`, `insights`, `picks`, `sectors`, `analysis`, `table`, `accuracy`, `charts`, `disclaimer`, `footer`, `error`, `sector`, and `lang`.

## Adding a new translation key

<Steps>
  <Step title="Add the key to translations.ts">
    Open **`dashboard/src/lib/translations.ts`** and add your key to both the `en` and `es` objects. Place it in the logical group it belongs to:

    ```typescript
    en: {
      // ... existing keys ...

      // Toolbar
      "toolbar.export": "Export CSV",
    },
    es: {
      // ... existing keys ...

      // Toolbar
      "toolbar.export": "Exportar CSV",
    }
    ```

    Both language entries are required. If you omit the `es` key, the `t()` function falls back to the `en` value automatically — but it's better to provide a real translation.
  </Step>

  <Step title="Use the translation in your component">
    Import and call the `useLanguage` hook in the component where you need the string:

    ```typescript
    import { useLanguage } from "@/hooks/use-language"

    export function Toolbar() {
      const { t } = useLanguage()

      return (
        <button>{t("toolbar.export")}</button>
      )
    }
    ```

    The `t()` function looks up the key in the current language's translation map and falls back to the `en` value (then the raw key) if the translation is missing.
  </Step>
</Steps>

## The useLanguage hook

The hook is defined in **`dashboard/src/hooks/use-language.tsx`** and exposes this interface:

```typescript
interface LanguageContextValue {
  lang: Language           // Current language: "en" | "es"
  setLang: (lang: Language) => void  // Switch language and persist to localStorage
  t: (key: string) => string         // Translate a key
  showPicker: boolean      // Whether the language picker modal is visible
  dismissPicker: () => void          // Close the picker
}

export function useLanguage(): LanguageContextValue
```

The selected language is persisted to `localStorage` under the key `tododeia-lang-v2`. On first load, if no preference is stored, the language picker modal appears automatically.

## What the Translation Agent handles

Step 8b of SKILL.md defines exactly which fields in `report.json` the Translation Agent translates into Spanish:

| Scope | Fields translated |
|---|---|
| Top level | `executive_summary`, `strategy_summary` |
| `macro_environment` | `summary`, `key_factors[]` |
| `cross_sector_insights[]` | `insight`, `implication` |
| Top level | `warnings[]` |
| `historical_accuracy` | `notable` |
| Per sector | `sector_summary`, `top_pick_reasoning` |
| Per asset | `reasoning`, `key_news[]`, `social_highlights[]` |

The Translation Agent explicitly preserves all numbers, tickers, prices, dates, percentages, asset names, symbols, URLs, and enum values (such as `"bullish"`, `"buy"`, `"high"`) exactly as-is.

You do not need to add these fields to `translations.ts` — they are handled automatically at runtime and written to **`dashboard/public/data/report-es.json`**.
